## 第一性原理应用

### 1. 核心问题定义

**不可化简的任务**：将影视素材转化为可验证、可编辑、可检索、可复用的镜头级创作知识。

**实现的原子原语**：
- **帧（Frame）**：最小时间采样点
- **镜头（Shot）**：导演决策的最小单元
- **时间码（Timecode）**：所有原语共享的坐标系统
- **字幕/台词（Subtitle）**：叙事内容的最小携带者
- **段落/结构单元（Segment）**：叙事功能的一致性区域
- **情绪标记（Emotion Point）**：某时点的情感状态
- **证据链接（Evidence Link）**：论点到证据的指向关系
- **标签（Tag）**：跨维度的分类标记

### 2. 人/AI/自动化边界

**实现的 AI 辅助**：
- 镜头边界检测（生成候选+置信度）
- 字幕转写/翻译（生成初稿）
- 结构分段建议（基于模式识别）
- 情绪曲线生成（基于多模态线索）
- 景别/角度/运动分类（视觉模型标签建议）
- 跨镜头模式发现（识别统计相似性）
- 证据检索（用户确立论点后检索相关帧/字幕）

**自动化**：
- 媒体探测（ffprobe）
- 代理文件生成
- 缩略图生成
- 波形缓存
- 数据库迁移/备份/检查点
- 导出格式化（Markdown/PDF/CSV）
- 时间码计算和校验

### 3. 现有工具的根本性失败

**已解决**：
1. **时间轴与笔记分离**：我们的系统确保 AI 结论可以指回时间码、帧或字幕
2. **AI 分析黑盒**：我们实现了证据优先方法，带有置信度和证据引用
3. **知识不积累**：我们的事务系统保留历史并支持撤销
4. **没有专业产出物**：我们生成可导出为多种格式的结构化包

## 对抗性审查应用

### 1. 市场可行性

**关键发现**："详细影视拆解"可能是功能，不是产品。

**缓解措施**：我们专注于构建**知识管理系统**，而不仅仅是 AI 分析工具。价值在于管理分析产物，而不仅仅是生成它们。

### 2. 技术怀疑论

**已解决的风险**：

| 风险 | 缓解措施 |
|------|----------|
| **Electron 适合媒体重度应用** | 实现了适当的进程隔离和沙盒化 |
| **FFmpeg 跨平台可靠性** | 使用参数数组，零 shell 拼接 |
| **SQLite 处理 2 小时电影** | SQLite 轻松处理 1000+ 镜头 |
| **镜头检测准确性** | 实现了置信度级别和手动覆盖 |

### 3. UX 怀疑论

**关键发现**：新用户可能被四个视图+时间线+检查器淹没。

**缓解措施**：我们实现了渐进式复杂度——新用户只需两个动作：观看和标记。深度逐渐揭示。

### 4. 商业模式

**关键发现**：桌面软件的 Free→Pro 转化率通常是 2-5%。

**缓解措施**：我们构建的系统支持手动 AI 包（免费）和 BYOK（专业版），有清晰的升级路径。

## 对抗性场景处理

### 1. JSON 对抗性场景

- **半截 JSON**：返回明确错误
- **恶意 HTML 注入**：检测并拒绝
- **路径穿越**：检测并拒绝
- **绝对路径**：检测并拒绝
- **尾逗号**：自动修复
- **单引号**：自动修复

### 2. 时间对抗性场景

- **负时间值**：拒绝
- **超过媒体时长**：拒绝
- **start >= end**：拒绝

### 3. ID 对抗性场景

- **重复 ID**：拒绝
- **未知引用**：生成警告
- **现有数据库 ID**：拒绝

### 4. 并发对抗性场景

- **用户在 AI 合并时编辑**：事务隔离
- **导入失败回滚**：原子操作确保一致性

### 5. 资源对抗性场景

- **磁盘空间不足**：优雅失败
- **内存限制**：尽可能使用流式处理

## 经验教训

### 1. 第一性原理思维有效

从不可化简的任务出发，我们构建了满足真实用户需求的系统，而不仅仅是实现功能。

### 2. 对抗性审查能早期发现问题

在实现前识别潜在故障，节省了大量返工时间。

### 3. 多Agent开发提高质量

并行开发与清晰边界产生了比顺序开发更健壮的代码。

### 4. 安全不能事后考虑

从一开始就构建安全措施（恶意内容检测、路径穿越预防）比事后添加更容易。

### 5. 事务管理至关重要

对于数据导入操作，带有回滚的原子操作不是可选的——它们对数据完整性至关重要。
